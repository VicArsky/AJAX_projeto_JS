Promises

Promises ( Promessas) são uma forma de lidar com tarefas que demoram um tempo para
terminar em JavaScript, como buscar dados de um servidor. Uma Promise é como
uma promessa de que algo vai acontecer no futuro.

Cumprir a promessa (resolve): Tudo deu certo e ela entrega o resultado.

Quebrar a promessa (reject): Algo deu errado, e ela diz o que deu errado.

Exemplo:

function buscarDados() {
return new Promise((resolve, reject) => {
console.log("Buscando dados...");
setTimeout(() => {
const sucesso = true;
if (sucesso) {
resolve("Dados encontrados!");
} else {
reject("Erro ao buscar os dados!");
}
}, 2000);
});
}
buscarDados()
.then((mensagem) => {
console.log(mensagem);
})
.catch((erro) => {
console.log(erro);
});
console: Dados encontrados!
ou
Erro ao buscar os dados!

Vantagens de usar Promises:

  • Facilita o trabalho com operações assíncronas: Ajuda a lidar melhor com código assíncrono, como carregar dados de um servidor, evitando o uso de muitos callbacks.


  • Melhor organização do código: Com Promises, o código fica mais limpo e fácil de entender, usando métodos como .then() e .catch() para lidar com sucesso e erro.


  • Encadeamento: Permite que você faça várias operações assíncronas em sequência, organizando melhor as etapas de execução.


Desvantagens de usar Promises:

  • Curva de aprendizado: Pode ser um pouco difícil de entender no começo, especialmente se você não está acostumado a lidar com código assíncrono.


  • Complexidade em casos de múltiplas Promises: Quando você precisa esperar por várias Promises ao mesmo tempo, o código pode ficar mais complicado de gerenciar, apesar do Promise.all ajudar.


  • Mais processamento: Em alguns casos, o uso de Promises pode adicionar uma leve sobrecarga de processamento em comparação a funções simples.
